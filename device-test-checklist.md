# VoiceTodo 真机测试 Checklist

按风险从高到低排列。每个场景都来自真机前静态审查中无法完全确认的分支；已判定为必须修复的代码问题不放入本清单。

## 1. App Group 真实签名与 Widget 共享数据

**操作步骤**

1. 删除真机上的 VoiceTodo，并移除所有 VoiceTodo Widget。
2. 重新安装当前构建。
3. 完成引导，创建一条今天待办。
4. 添加桌面小/中/大 Widget 和锁屏 Widget。
5. 等待 Widget 首次刷新，必要时重新添加一次 Widget。

**预期结果**

- 主 App 创建的数据能在 Widget 中出现。
- Widget 不显示旧沙盒数据。
- App 和 Widget 的数据状态一致。
- Widget 不因共享数据库不可读而白屏。

## 2. Widget 打勾后主 App 状态同步

**操作步骤**

1. 在 App 内创建一条今天普通待办。
2. 添加桌面 Widget。
3. 将 App 放到后台。
4. 在 Widget 上点击待办完成按钮。
5. 立即回到主 App 查看同一条待办状态。
6. 再创建一条 daily recurring 待办，在 Widget 勾选后回 App 查看今天 occurrence。

**预期结果**

- Widget 点击后自身刷新，普通待办从 Widget 中消失或显示为已完成。
- 主 App 回到前台后状态刷新为已完成。
- daily recurring 只完成今天 occurrence，不影响未来日期。

## 3. 主 App 与 Widget 同时写入同一条待办

**操作步骤**

1. 在 App 内创建一条今天待办，并添加 Widget。
2. 保持 App 在前台首页。
3. 快速在 Widget 点完成，同时在 App 内也点完成/取消完成。
4. 切后台再回前台，必要时杀掉 App 重启。

**预期结果**

- App 不崩溃。
- Widget 不白屏。
- 最终 App 与 Widget 对同一条待办的完成状态一致。

## 4. 录音中被来电、Siri 或闹钟打断

**操作步骤**

1. 打开 VoiceTodo，确认麦克风和语音识别权限均已允许。
2. 点击“开始录音”，持续说一段 5 秒以上的话。
3. 用另一台设备拨打电话，或触发 Siri/闹钟造成音频中断。
4. 结束中断后回到 VoiceTodo。
5. 再次点击录音按钮，并尝试保存一条新待办。

**预期结果**

- App 不崩溃，不停留在无法操作的录音状态。
- 本次录音被明确取消，不自动保存 partial transcript。
- 后续再次录音可以正常开始、停止、进入确认页。

## 5. 录音中锁屏、切后台或下拉控制中心

**操作步骤**

1. 点击“开始录音”。
2. 说一句待办，例如“明天上午买牛奶”。
3. 分别测试锁屏 10 秒、切到其他 App 10 秒、下拉控制中心 5 秒。
4. 回到 VoiceTodo。
5. 尝试再次开始录音并保存。

**预期结果**

- App 不长时间卡在录音中或处理中。
- 回到 App 后能重新开始录音。
- 不产生错误待办。
- 不丢失已经保存成功的待办。

## 6. 手动停止录音的最终识别回调

**操作步骤**

1. 分别录制 5 秒、30 秒、60 秒三段语音。
2. 每段都手动点击停止。
3. 等待确认页或空结果提示。
4. 每次完成后再次开始下一段录音。

**预期结果**

- 点击停止后几秒内退出录音态。
- 不一直显示 processing/recording。
- 确认页或空结果提示能正常出现。
- 下一次录音能正常开始。

## 7. 连续停止/重录与音频会话释放

**操作步骤**

1. 连续执行“开始录音 -> 说一句话 -> 停止录音 -> 取消确认页”10 次。
2. 插入/断开蓝牙耳机或有线耳机后再重复 3 次。
3. 观察系统麦克风指示和 App 录音按钮状态。

**预期结果**

- 录音按钮状态与系统麦克风占用一致。
- 不出现按钮已停止但麦克风仍被占用。
- 下一次录音不会因为音频会话未释放而失败。

## 8. Onboarding 权限弹窗返回后的 UI 状态

**操作步骤**

1. 删除 App 后重新安装。
2. 进入 onboarding 麦克风权限页，点击授权，分别测试允许和拒绝。
3. 进入语音识别权限页，点击授权，分别测试允许和拒绝。
4. 观察权限卡片、按钮 loading 和下一步按钮。

**预期结果**

- 系统弹窗关闭后 loading 消失。
- 权限状态卡正确更新。
- 拒绝权限时能看到去设置入口。
- UI 不会卡在不可点击状态。

## 9. 旧数据迁移后显示位置

**操作步骤**

1. 使用包含旧版 dueHint 数据的安装包升级到当前版本，或用测试数据模拟只有 dueHint、没有 dueDate 的待办。
2. 打开首页月历。
3. 检查“今天/明天/周五”等旧待办是否出现在对应日期。

**预期结果**

- 旧待办不会全部落到未安排区。
- 日期解析后的待办出现在对应月历日期下。
- App 不因迁移失败崩溃。

## 10. Widget/AppIntent 写入后 App 刷新失败的恢复

**操作步骤**

1. 在 Widget 上完成一条待办。
2. 回到 App，如果状态没有立刻刷新，杀掉 App 后重启。
3. 重复测试普通待办和 recurring occurrence。

**预期结果**

- 正常情况下回前台直接同步。
- 如果首次刷新失败，重启后状态正确。
- 不出现 App 和 Widget 长期不一致。
